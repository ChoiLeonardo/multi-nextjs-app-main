
아래는 **유도탄 ‘조립 후 통합시험(Integrated System Test, IST)’ 단계**에서  
**DFT + SPC + AI + 의사결정**이 **하나의 시스템으로 결합된 전체 구조(End-to-End Architecture)** 제안입니다.

이 구조는

- ✔ 실시간(20 ms) 운용 가능
    
- ✔ 방산·고신뢰 SW 인증 논리 대응
    
- ✔ “이상 탐지 → 원인 추적 → 운용 판단”까지 일관되게 연결  
    되도록 설계되었습니다.
    

---

# 1. 통합시험 전체 구조 개요 (Top-Level)

```
┌──────────────────────────────────────────────┐
│         Integrated System Test (IST)      ㅂ    │
├──────────────────────────────────────────────┤
│ ① DFT 기반 시험·계측 계층                     │
│ ② 실시간 데이터 수집·정규화 계층              │
│ ③ SPC Fast Path (20 ms)                       │
│ ④ SPC Slow Path (누적)                        │
│ ⑤ AI 상호작용 분석 계층                      │
│ ⑥ 의사결정·조치 계층                          │
│ ⑦ 시각화·이력·형상 관리                       │
└──────────────────────────────────────────────┘
```

---

# 2. ① DFT 기반 시험·계측 계층 (통합시험 전용)

### 역할

- **상호작용 관측 가능화**
    
- 단품 정상 여부가 아닌 **결합 상태 가시화**
    

### 구성 요소 (최소 필수)

|영역|DFT 신호|
|---|---|
|전기|전압 강하, 리플, 전류|
|시간|INS–탐색기 타임 오차|
|데이터|CRC error, 지터|
|열|구성품별 요약 온도|
|기계|엔진 진동 RMS|
|내부 상태|AGC, INS bias, RPM 안정도|

👉 **이 계층이 없으면 SPC·AI는 ‘눈이 없음’**

---

# 3. ② 데이터 수집·정규화 계층

### 역할

- 시험환경·장비 차이 제거
    
- 모든 분석 계층에 동일 입력 제공
    

### 처리 내용

- 단위/스케일 정규화
    
- 온도·전압 보정
    
- 타임스탬프 정렬
    
- 메타데이터 결합 (형상, LOT, 기령)
    

---

# 4. ③ SPC Fast Path (20 ms 실시간)

### 목적

- **즉시 위험 차단**
    
- Fail-safe 역할
    

### 적용 기법 (O(1) 연산)

|기법|대상|
|---|---|
|Z-score|전압, 전류|
|EWMA|지터, 진동|
|임계치|타임오차|

### 출력

```json
{
  "fast_flag": "WATCH",
  "violations": ["Bus ripple rising"]
}
```

---

# 5. ④ SPC Slow Path (누적·정밀)

### 목적

- **구조적 이상 탐지**
    
- 상호작용 안정성 판단
    

### 입력

- 0.5~2초 누적 데이터
    

### 적용 기법

|기법|의미|
|---|---|
|Hotelling’s T²|다변량 결합 이상|
|PCA SPC|상관 구조 변화|
|EWMA/CUSUM|점진 이동|

### 출력

```json
{
  "spc_status": "ANOMALY",
  "cause_hint": "Engine–INS coupling"
}
```

---

# 6. ⑤ AI 상호작용 분석 계층 (보조 판단자)

### AI의 역할 재정의

> **“이상이 왜 발생했는지 설명”**

---

## 6.1 적용 모델 구성

### (1) Autoencoder (정상 결합 패턴 학습)

- 입력: 탐색기 + INS + 엔진 + 인터페이스
    
- 출력: 재구성 오차
    

### (2) PCA 기여도 분석

- 어떤 구성품 간 결합이 깨졌는지 표시
    

---

### 출력 예

```json
{
  "ai_score": 0.81,
  "dominant_interaction": "Engine vibration → INS drift"
}
```

👉 **AI 단독 판정 금지**

---

# 7. ⑥ 의사결정·조치 계층 (Rule + Human-in-the-loop)

### 통합 판정 매트릭스

| Fast SPC | Slow SPC | AI  | 결정     |
| -------- | -------- | --- | ------ |
| OK       | OK       | 정상  | 통과     |
| WATCH    | OK       | 이상  | 조건부 통과 |
| ALERT    | 이상       | 이상  | 격리     |
| OK       | WARNING  | 정상  | 재시험    |

---

### 조치 예시

- 시험 중단
    
- 조건부 통과 (RPM 제한)
    
- 정밀 점검 요청
    

---

# 8. ⑦ 시각화·이력·형상 관리 계층

### 주요 기능

- 통합 관리도
    
- 상호작용 Heatmap
    
- 시험 이력 추적
    
- 형상 변경 시 Baseline 무효화
    

---

# 9. 실시간 운용 관점 구조 요약

```
20 ms:
  DFT → Fast SPC → 안전 플래그

1~2 s:
  누적 → Slow SPC → AI 분석 → 판정 갱신
```

# 10. 방산·고신뢰 인증 대응 논리

|요소|대응 논리|
|---|---|
|판정 책임|SPC 규칙|
|AI|참고 정보|
|재현성|DFT 기반|
|변경 관리|형상 통제|

---

# 11. 한 문장 요약 (설계 철학)

> **통합시험은 “부품의 정상 여부”가 아니라  
> “결합된 시스템의 안정성”을 판단하는 단계이며,  
> DFT는 눈, SPC는 기준, AI는 해석자 역할을 한다.**
