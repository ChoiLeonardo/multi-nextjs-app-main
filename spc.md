- **시스템 예시 (RAAD-LLM)**: RAAD-LLM은 실제 구현 사례로, 주조 공장 설비 데이터를 이용해 “적응형 이상 탐지”를 수행했습니다.
- 이 시스템은 
- 1. 분석 레이어_에서 통계적 공정관리(SPC) 및 DFT를 사용해 이상치를 선별하고,
  2. RAG 레이어_로 CSV 기반 지식베이스에서 기준 값을 검색하며,
  3. LLM 레이어_로는 Llama 모델을 이용해 질의 응답형 대화로 이상 여부를 판단했습니다
- 즉, 수치 계산과 도메인 지식 검색은 전통적 방법으로 처리하고, LLM은 그 결과를 기반으로 의사결정 질문에 답변하도록 한 것입니다.





유도탄 점검(Missile Health Check / Acceptance Test / Periodic Inspection)과 같은 **고신뢰·안전중심 응용 도메인**에서  
**통계적 공정관리(SPC)**와 **DFT(Design For Testability)**를 결합해 이상치를 선별·관리하는 방안은  
“**이상 신호의 조기 탐지 + 원인 추적 가능성 확보**”가 핵심 목표가 됩니다.

아래는 **실무 적용 관점**에서 바로 활용 가능한 구조로 정리한 제안입니다.

---

## 1. 기본 개념 정리 (도메인 특화 관점)

### ① 통계적 공정관리(SPC)

- 목적: **정상 상태에서의 변동(Noise)**과 **이상 원인(Assignable Cause)**을 구분
    
- 유도탄 점검 적용 시:
    
    - 개별 장비의 절대 성능보다 **동일 형상·동일 LOT·동일 공정 간의 편차**에 집중
        
    - “불량 판정”보다 **이상 징후(Early Warning)** 탐지가 중요
        

### ② DFT (Design For Testability)

- 목적: **시험 중 내부 상태를 관측 가능하게 설계**
    
- 유도탄 점검 적용 시:
    
    - 시험 포인트 삽입
        
    - 내부 신호/상태의 정량화
        
    - 시험 반복 시 일관된 데이터 확보
        

👉 **핵심 포인트**

> SPC는 “이상인지 아닌지”를 판단하고,  
> DFT는 “왜 이상인지”를 추적 가능하게 만든다.

## 2. 전체 아키텍처 개요

```
[DFT 기반 시험 설계]
        ↓
[정량화된 시험 데이터 수집]
        ↓
[SPC 기반 정상 공정 모델링]
        ↓
[이상치 탐지 (단계별)]
        ↓
[원인 분류 / 조치 전략]
```

## 3. DFT 관점의 시험 설계 방안 (이상치 관리의 출발점)

### 3.1 시험 신호/변수 설계 원칙

|항목|설계 원칙|
|---|---|
|시험 신호|단일 기능 자극보다 **분리 가능한 자극 패턴**|
|측정 변수|합격/불합격이 아닌 **연속형 수치 데이터**|
|반복성|동일 조건 반복 시 분산이 작도록 설계|
|민감도|고장 모드별 반응 차이가 드러나도록|

**예시**

- RF 송수신 점검
    
    - 단순 PER →  
        ▶ SNR, EVM, IQ imbalance, AGC settling time 분리 측정
        
- 관성항법장치
    
    - PASS/FAIL →  
        ▶ 바이어스, 스케일 팩터, Allan variance 파라미터
        

---

### 3.2 DFT 요소 삽입 예시

|DFT 요소|이상치 관리에 주는 효과|
|---|---|
|내부 상태 레지스터 Readout|Black-box → Gray-box|
|Self-Test Pattern|반복 가능 기준 데이터 확보|
|Loop-back 경로|외부 환경 영향 분리|
|시험 모드 고정 클럭|시간축 변동 제거|

---

## 4. SPC 기반 이상치 탐지 전략 (단계별)

### 4.1 1단계: 기준선(Baseline) 설정

**집단 정의가 핵심**

- 동일:
    
    - 형상 (Block / Version)
        
    - 제조 LOT
        
    - 시험 환경 (온도, 장비)
        
    - 시험 시점(노후화 영향 분리)
        

**통계 모델**

- 평균(μ), 표준편차(σ)
    
- 필요 시 Robust 통계 (Median, MAD)
    

---

### 4.2 2단계: 관리도 적용 전략

#### (1) X̄–R / X̄–S 관리도

- 반복 시험 가능한 항목
    
- 아날로그 특성 (전압, 주파수, 지연)
    

#### (2) I–MR 관리도

- 반복 시험 불가 or 단일 샘플
    
- 현장 점검, 주기 점검
    

#### (3) 다변량 관리도 (Hotelling’s T²)

- RF, INS, 전원 등 **상호 연관된 변수**
    
- 단일 항목 정상이나 **조합상 이상 탐지 가능**
    

---

### 4.3 3단계: 이상치 판정 기준 (Fail ≠ 이상)

|구분|의미|
|---|---|
|규격 이탈|명백한 불합격|
|관리 한계 초과|공정 이상 가능성|
|추세 이상|잠재 고장 전조|
|분산 증가|환경/노후화 영향|

👉 **운용 포인트**

- **3σ 초과 = 즉시 분석**
    
- **2σ 연속 추세 = 예비 경보**
    
- **LOT 단위 이동 = 공정 문제**
    

---

## 5. 이상치 분류 및 관리 로직

### 5.1 이상치 유형 분류

|유형|특징|대응|
|---|---|---|
|단발성 이상|단일 샘플 튀는 값|재시험|
|구조적 이동|평균 이동|공정/설계 점검|
|분산 증가|불안정성|환경·노후 분석|
|상관 이상|다변량에서만 탐지|시스템 레벨 점검|

---

### 5.2 DFT + SPC 연계 원인 추적

**예시: RF 송신 출력 이상**

1. SPC에서 출력 평균 이동 탐지
    
2. 내부 DFT 포인트 확인
    
    - PA bias 전압
        
    - 온도 센서
        
3. 원인 분류
    
    - 설계 문제 / 소자 열화 / 시험환경
        

👉 **결과**

- 단순 “불량” →  
    **“PA 바이어스 드리프트로 인한 성능 열화 가능성”**

## 6. 운영 관점의 이상치 관리 체계

### 6.1 점검 단계별 전략

|단계|SPC 활용 포인트|
|---|---|
|생산/조립|공정 안정성 확인|
|출하 시험|초기 이상 선별|
|주기 점검|노후화 추적|
|임무 전 점검|위험 조기 제거|

---

### 6.2 데이터 축적 기반 고도화

- LOT 이력 기반 기준선 자동 갱신
    
- 기령(Age)별 분포 분리
    
- 향후:
    
    - SPC → **통계 + ML 혼합 이상탐지**
        
    - DFT 데이터 → **고장 모드 학습 데이터**
        

---

## 7. 정리 (실무적 핵심 메시지)

1. **DFT는 이상치의 ‘관측 가능성’을 만든다**
    
2. **SPC는 이상치의 ‘통계적 의미’를 판단한다**
    
3. Fail/Pass 중심 점검 → **분포·추세 중심 관리**로 전환
    
4. 유도탄 점검에서는
    
    - “고장 탐지”보다
        
    - **“고장으로 가는 경향의 조기 차단”이 핵심
